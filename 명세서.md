# 을지대학교 RAG 시스템 구축 프로젝트 명세서

> **이 문서의 대상**: Claude Code (구현 담당)
> **작성 목적**: 프로젝트의 전체 그림, 결정된 기술 스택, 다음 작업의 우선순위를 한 문서로 전달
> **현재 단계**: 데이터 전처리 완료 → 인덱싱·검색·생성 파이프라인 구현 단계

---

## 1. 프로젝트 개요

### 1.1 무엇을 만드는가
을지대학교 학생·교직원이 자연어 질문으로 학교 관련 정보(학칙, 학사일정, 장학금, FAQ, 강의평가 등)를 정확하고 출처 있는 답변으로 받을 수 있는 **한국어 특화 RAG 챗봇**.

### 1.2 왜 RAG인가
- LLM 단독은 학교 내부 규정·일정 같은 도메인 데이터를 모름 → 환각 위험
- 데이터가 자주 갱신됨(학기별 일정, 강의평가) → 파인튜닝보다 RAG가 비용·속도 면에서 우월
- 답변에 출처 인용이 필수(학칙·졸업요건 같은 공식 정보)

### 1.3 사용자 질문 예시
- "의정부캠퍼스 간호학과 졸업하려면 몇 학점 필요해?" (학칙·학사정보)
- "수강신청 언제 시작해?" (학사일정 + 시간 필터링)
- "글로벌문화영어 시험 어렵게 나와?" (강의평가 - 시험 섹션)
- "장학금 신청은 어디서 해?" (장학금 카테고리)
- "기숙사 식권 몇 장 줘?" (FAQ)

### 1.4 성공 기준
- **정답률(faithfulness)** 0.85 이상 (RAGAS 기준)
- **검색 recall@5** 0.90 이상
- **응답 시간(p95)** 3초 이내
- **환각 검증 통과율** 0.80 이상 (Groundedness Check `grounded` 비율)

## 3. 결정된 기술 스택

### 3.1 핵심 모델 (모두 Upstage Solar)

| 역할 | 모델명 | API 엔드포인트 / 비고 |
|---|---|---|
| **문서 임베딩 (인덱싱)** | `solar-embedding-1-large-passage` | `https://api.upstage.ai/v1/solar/embeddings` |
| **쿼리 임베딩 (검색)** | `solar-embedding-1-large-query` | 같은 엔드포인트, 모델명만 변경 |
| **답변 생성 LLM** | `solar-pro` (정확도 우선) / `solar-mini` (속도 우선) | `https://api.upstage.ai/v1/solar/chat/completions` |
| **환각 검증** | `UpstageGroundednessCheck` | `langchain_upstage` 패키지 |

**임베딩 차원**: 4,096
**LLM 컨텍스트**: 32K
**임베딩 권장 토큰**: ≤512 (이미 전처리에서 91.7% 만족)

### 3.2 검색·평가 스택

| 컴포넌트 | 선택 | 이유 |
|---|---|---|
| **RAG 프레임워크** | **AutoRAG** (`Marker-Inc-Korea/AutoRAG`) | 한국 개발, Solar 공식 통합, AutoML 방식 자동 최적화 |
| **벡터 DB** | **Qdrant** (자체 호스팅) | sparse+dense 네이티브 통합, 메타데이터 필터링 강력 |
| **BM25 토크나이저** | **ko_okt** (1순위), ko_kkma (2순위) | AutoRAG 한국어 벤치마크 1위 (F1 0.798) |
| **리랭커** | **`dragonkue/bge-reranker-v2-m3-ko`** | 한국어 최적화 BGE 리랭커 (정확도 0.91) |
| **검색 융합** | **Reciprocal Rank Fusion (RRF) k=60** | 점수 정규화 불필요, 91% recall@10 |
| **평가 도구** | **RAGAS** | faithfulness, answer_relevancy, context_precision |
| **백엔드 프레임워크** | **FastAPI** | 비동기 처리, OpenAPI 자동 문서화 |
| **프론트엔드 (MVP)** | **Streamlit** (`autorag run_web` 활용) | 즉시 데모 가능 |
| **프론트엔드 (운영)** | **Next.js** | 고도화 시 |
| **캐시** | **Redis** + 시맨틱 캐싱 | LLM 비용 30~68% 절감 |
| **모니터링** | **Langfuse** 또는 자체 로깅 | 답변 품질·지연 추적 |

## 4. 시스템 아키텍처

### 4.1 전체 흐름

```
[사용자 질문]
    ↓
[1. 쿼리 라우터] - 카테고리·캠퍼스 키워드 추출
    ↓
[2. 쿼리 변환 (선택)] - HyDE / Query Decompose
    ↓
[3. 하이브리드 검색]
    ├─ Solar Query 임베딩 → Qdrant Dense 검색
    └─ Okt 토큰화 → BM25 Sparse 검색
    → RRF k=60 융합 → top-30 후보
    ↓
[4. 메타데이터 필터링] - campus, source_collection 일치 항목 우선
    ↓
[5. 크로스 인코더 리랭킹] - bge-reranker-v2-m3-ko → top-5
    ↓
[6. 컨텍스트 조립] - 청크 + 메타데이터 인용 정보
    ↓
[7. Solar Pro 답변 생성]
    ↓
[8. UpstageGroundednessCheck] - 환각 검증
    ├─ grounded → 사용자에게 전달 + 출처 표시
    ├─ notSure → 경고 표시 후 전달
    └─ notGrounded → 재검색 (top-k 5→10) 후 재시도, 실패 시 거절
    ↓
[9. 응답 + 출처 인용]
    ↓
[10. RAGAS 로깅] - 일별 품질 모니터링
```

### 4.2 9개 컬렉션 → 라우팅 전략

| 컬렉션 | 검색 가중치 | 특이사항 |
|---|---|---|
| `학칙_조항` | BM25 0.5 / Dense 0.5 | 조항 번호(`제31조`) 정확 매칭 중요 |
| `FAQ` | BM25 0.6 / Dense 0.4 | Q+A 쌍 통째 유지, 키워드 매칭 강함 |
| `시설_연락처` | BM25 0.7 / Dense 0.3 | 전화번호·학과명 정확 매칭 |
| `장학금` | BM25 0.4 / Dense 0.6 | 의미 검색이 중요 |
| `학사정보` | BM25 0.4 / Dense 0.6 | 절차 가이드형 |
| `학사일정` | BM25 0.5 / Dense 0.5 | **+ 날짜 메타데이터 필터링 필수** |
| `강의평가` | BM25 0.3 / Dense 0.7 | 의미 검색 중심 + 강의명 정확 매칭 |
| `학과정보` | BM25 0.5 / Dense 0.5 | |
| `교육과정` | BM25 0.5 / Dense 0.5 | `low_confidence` 표시 |
| `기타` | BM25 0.5 / Dense 0.5 | |

#### Task 4.3: Groundedness Check 통합 ★중요
**파일**: `src/generation/groundedness.py`

요구사항:
- `langchain_upstage.UpstageGroundednessCheck` 사용
- 3가지 위치에서 활용:
  1. 답변 생성 직후 1차 검증
  2. `notGrounded` 시 재검색 트리거 (top-k 5→10, 1회만)
  3. 일별 KPI 로깅

```python
from langchain_upstage import UpstageGroundednessCheck

groundedness = UpstageGroundednessCheck()

async def verify_answer(context: str, answer: str) -> str:
    """returns: 'grounded' | 'notGrounded' | 'notSure'"""
    result = await groundedness.ainvoke({
        "context": context,
        "answer": answer,
    })
    return result
```

#### Task 4.4: Self-Corrective RAG 흐름
**파일**: `src/pipeline/rag_pipeline.py`

```python
async def rag_query(query: str) -> dict:
    # 1차 시도
    candidates = await hybrid_search(query, top_k=30)
    candidates = await rerank(query, candidates, top_k=5)
    answer = await generate(query, candidates)
    grounded = await verify_answer(format_context(candidates), answer)

    if grounded == "notGrounded":
        # 2차 시도: top-k 확대 + HyDE
        expanded_query = await hyde_expand(query)
        candidates = await hybrid_search(expanded_query, top_k=50)
        candidates = await rerank(query, candidates, top_k=10)
        answer = await generate(query, candidates)
        grounded = await verify_answer(format_context(candidates), answer)

        if grounded == "notGrounded":
            return {
                "answer": "제공된 자료에서 해당 정보를 찾을 수 없습니다.",
                "grounded": False,
                "sources": [],
            }

    return {
        "answer": answer,
        "grounded": grounded == "grounded",
        "sources": [c["metadata"] for c in candidates],
    }
```
